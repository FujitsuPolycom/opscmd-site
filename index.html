How it is used

Agents ask. The gateway decides.

OpsCMD turns infrastructure actions into typed, policy-aware tool calls. Low-risk reads can run quickly. High-risk changes pause for approval before credentials are resolved.

1

Request

A human, Pi, Hermes, or another agent asks Core to run a named tool with typed args.

2

Gate

Core checks token role, tool policy, network scope, secret scope, and approval rules.

3

Run

The worker executes the job with bounded timeout, redaction, and server-side secrets.

4

Audit

Jobs, events, approvals, actors, outputs, and failures are recorded for review.
